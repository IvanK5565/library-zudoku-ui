​Software Requirement Specification

​Вступ

​Призначення документа

Цей документ визначає функціональні та нефункціональні вимоги до веб-додатку "Електронний щоденник", розробленого з використанням фреймворків Next.js та React. Метою є створення інструменту для обліку навчальної діяльності учнів, що дозволяє вчителям виставляти оцінки, а батькам і учням — переглядати їх.

​Область застосування

Система призначена для середніх загальноосвітніх навчальних закладів. Основні користувачі:

  • Адміністратори школи
  • Вчителі
  • Учні
  • Батьки

​Визначення, абревіатури та скорочення

  • Next.js — фреймворк для SSR/SSG React-додатків
  • SSR — Server-Side Rendering
  • CRUD — Create, Read, Update, Delete
  • JWT — JSON Web Token для авторизації
  • REST API — архітектурний стиль інтерфейсів взаємодії клієнт-сервер

​Загальний опис

​Перспективи продукту

Електронний щоденник є окремим веб-додатком із підтримкою клієнт-серверної архітектури. Веб-клієнт реалізується за допомогою React та Next.js, серверна частина — через REST API (можливо з Node.js/Express або стороннім API).

​Функції продукту

  • Реєстрація та авторизація користувачів (JWT)
  • Панель вчителя: створення класів, виставлення оцінок, перегляд статистики
  • Панель учня: перегляд оцінок, розкладу, завдань
  • Панель батьків: перегляд оцінок своїх дітей
  • Панель адміністратора: управління користувачами та класами

​Характеристики користувачів

  • Учні — базові навички користування ПК
  • Батьки — базові навички, доступ до даних дітей
  • Вчителі — середній рівень, можуть створювати записи
  • Адміністратори — високий рівень, повний доступ

​Обмеження

  • Підтримка сучасних браузерів (Chrome, Firefox, Safari, Edge)
  • SSR за допомогою Next.js
  • Використання бази даних (наприклад PostgreSQL або MongoDB)

​Припущення і залежності

  • Користувачі мають доступ до Інтернету
  • Серверна частина може бути реалізована окремою командою
  • Email-верифікація або інші механізми безпеки реалізуються окремо

​Функціональні вимоги

​Реєстрація та авторизація

  • F1.1: Користувач може зареєструватися (вказавши роль)
  • F1.2: Користувач може авторизуватись
  • F1.3: JWT токени для захисту маршрутів

​Панель вчителя

  • F2.1: Створення класів та предметів
  • F2.2: Додавання оцінок
  • F2.3: Коментарі до оцінок
  • F2.4: Завантаження домашніх завдань

​Панель учня

  • F3.1: Перегляд оцінок по датах та предметах
  • F3.2: Перегляд домашніх завдань
  • F3.3: Статистика успішності

​Панель батьків

  • F4.1: Прив’язка до учня (за кодом)
  • F4.2: Перегляд оцінок та коментарів вчителів

​Панель адміністратора

  • F5.1: Керування користувачами
  • F5.2: Призначення ролей
  • F5.3: Керування класами та розкладом

​Нефункціональні вимоги

​Продуктивність

  • Сторінки повинні завантажуватись менше ніж за 2 секунди

​Масштабованість

  • Підтримка до 1000 активних користувачів одночасно

​Безпека

  • Захист авторизації через HTTPS та JWT
  • Валідація форм на клієнті й сервері

​Зручність інтерфейсу

  • Інтуїтивний дизайн відповідно до ролі користувача
  • Адаптивний інтерфейс (mobile-first)

​Можливість розширення

  • Можливість додати модулі (чат, сповіщення)

​Інші вимоги

  • Використання TypeScript (рекомендовано)
  • Застосування компонентного підходу (React)
  • SSR/SSG для SEO та швидкості
  • CI/CD для автоматичного деплою (наприклад, Vercel)
